# OpenRead Branding

OpenRead is an open-source, BYOK translation extension for careful reading across languages. The brand should feel open, technical, trustworthy, and readable.

## Positioning

OpenRead is not a general AI assistant and not a closed translation service. It is a reader-owned translation layer for the open web.

Core associations:

- Open source and hackable
- Bring your own key
- Web reading first
- Bilingual by default
- International communication without subscription lock-in

## Slogan

Primary:

> Open the web. Read every language.

Chinese:

> 打开网络，读懂每一种语言。

Short form:

> Open web, open language.

## Logo Direction

The product icon should keep three ideas visible:

- An open book for reading.
- A paired-page structure for bilingual reading.
- A warm, easy-to-spot toolbar tile.

The icon should be direct rather than overly clever. Developer references such as `i18n`, brackets, terminals, and patches can appear in supporting graphics, but the browser extension icon should remain recognizable to non-developers at 16px and 32px. Small toolbar sizes should prefer purpose-drawn pixel masters over lacquer texture, thin outlines, letterforms, or dense page details.

## Color Tokens

Primary brand colors:

| Token | Hex | Use |
| --- | --- | --- |
| Ink | `#141413` | Text, primary controls, high-contrast marks |
| Charcoal | `#3D3D3A` | Secondary text and icon shadow shapes |
| Ivory | `#F0EEE6` | App canvas and quiet extension surfaces |
| Paper | `#FFFDF7` | Input and card surfaces |
| Terracotta | `#D97757` | Icon tile, active controls, brand accent |
| Umber | `#8F3F2B` | Book shadows and strong accent text |
| Oat | `#E8E6DC` | Muted surfaces, dividers, inactive controls |
| Sage | `#5F7F3F` | Success state |

The main accent is terracotta, not cyan or pink. The palette should feel warm, matte, editorial, and calm rather than glossy or neon.

## Icon Rules

- Keep the terracotta rounded square, ivory open book, and gold center spine as the stable core.
- Preserve enough contrast for Chrome toolbar sizes.
- Prefer crisp pixel shapes over glow, folded-paper details, dense page lines, decorative gradients, or metallic texture.
- Maintain separate 16px and 32px pixel masters when needed; the 16px master should optimize recognition, while the 32px master can carry slightly clearer book geometry.
- Avoid dense text inside the icon. At toolbar size, literal language glyphs often read as noise.
- Avoid copying Apple, Dia, Google Translate, or Immersive Translate visual systems.
- Avoid emoji, mascots, flags, globes, and excessive gradients.
- Prefer a calm technical tone over playful illustration.

## UI Tone

OpenRead UI should feel like a precise reading tool:

- Quiet surfaces.
- Clear provider controls.
- Copyable raw errors.
- Small status chips.
- No marketing-style hero treatment inside extension surfaces.

Page translation should inherit the page instead of visually branding the page. Brand colors belong mainly in extension UI, the icon, and small OpenRead-owned controls. Chrome toolbar visibility takes priority over high-fidelity material effects.

## Accessibility

- Icon-only controls need accessible labels.
- Brand colors must not be the only signal for success, failure, or progress.
- Keep status icons paired with localized text where space allows.
